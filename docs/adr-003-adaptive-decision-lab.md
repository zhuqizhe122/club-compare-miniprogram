# ADR-003：采用本地、证据感知的自适应决策实验室

- 状态：Accepted for planned implementation
- 日期：2026-07-24
- 关联：`specs/003-adaptive-decision-lab/`

## 背景

001/002 checkpoint 已证明“四题缩小范围 → 选择 2–3 个 → 对照 → 倾向”的最小闭环，但四题精确枚举匹配无法充分表达不可妥协条件、资料可靠性和现实压力。98 社的多数结构化字段来自原型推断；若直接将其用于“满足时间上限”等硬结论，会制造错误确定性。

## 决策

1. 将目标信息架构扩展为八页：
   `index → assessment → recommend → library → club → selection → compare → result`。
2. 评估采用确定性 8–14 题：8 道核心题后，仅为消除硬约束未知、区分前列候选或补低置信维度追加题。
3. 将硬约束与软偏好分离：
   - 硬约束只输出 PASS、FAIL、UNKNOWN；
   - 软偏好输出独立的 0–100 多准则匹配；
   - 证据置信度单独显示，不乘入偏好分。
4. 数据从对象级“待确认”演进为字段级 A–D/U evidence。
5. `prototype-inferred` 永远不能支持硬约束 PASS；相关字段匹配时仍为 UNKNOWN。
6. 深度比较增加确定性压力情境，只输出 RESILIENT、AT_RISK、UNKNOWN，不输出概率。
7. 保持原生微信小程序、本地数据、会话态、无网络、无登录、无新依赖和线下真实加入边界。

## 理由

- 三值逻辑避免把“没有证据证明冲突”误写成“已经证明满足”。
- 分开偏好分与证据置信度，避免资料最完整的社团自动变成“最适合”。
- 字段级 provenance 让报告中的每个关键事实可追溯。
- 确定性规则适合课程原型：可离线、可测试、可解释、无密钥和成本风险。
- 八页将评估、探索、详情、候选管理和深度决策拆成单一职责，解决旧 `list` 同页负载过高的问题。

## 被否决的方案

### 保留四页并继续叠加

`list` 同时承担问卷、推荐、搜索、候选管理和资料详情，会造成状态与认知负担，无法清楚展示 evidence 和压力测试。

### 用 evidence 权重直接折扣偏好分

这会混淆“我有多喜欢”与“我们有多确定”，导致用户无法判断低分来自不匹配还是资料不足。

### 将 UNKNOWN 当弱 PASS

这违反硬约束语义，尤其会让原型推断错误通过时间、费用等不可妥协条件。

### 接入 LLM 或远程推荐

会引入网络、隐私、密钥、成本、不可复现和不可解释风险，且对当前证据问题没有直接帮助。

## 后果

正面：

- 结论更诚实、可审计；
- 用户可以从推荐和主动探索两条路形成同一候选篮；
- 规则和报告能被固定夹具完整重算。

代价：

- 页面从四个迁移到八个；
- 98 社需逐字段审计，许多字段在短期内会保持 UNKNOWN；
- 会话状态需版本化并传播失效，测试面显著扩大。

## 迁移与撤销

迁移期间保留 001/002 checkpoint 回归，路由校验暂时接受批准路由的有序中间态。完成 003 收口时必须移除旧 `list` 注册并恢复八页精确断言。

若字段级证据无法在展示期限内形成，允许只实现评估与推荐的可评审薄片，但不得把后续六页或硬约束 PASS 伪装为完成。撤销专业版时回到 001/002 checkpoint，不改写其历史验收记录。
