## 微信开发者工具人工验收

### 数据、入口与双模式

- [ ] 首页能进入选择页，页面无报错；列表基础数据总数为 98 社。
- [ ] 选择页在同一页提供“帮我缩小范围”和“我已知道名字”两种模式，切换不新增页面。
- [ ] 引导模式完整显示并可依次回答 4 题：活动分类、方便时段、投入强度、互动方式。
- [ ] 第 4 题后显示不超过 6 项的 Top 6；每项显示 2–3 条可读理由和“待社团确认”。
- [ ] 搜索模式支持关键词搜索；分类标签可筛选，切回“全部”恢复对应完整结果。
- [ ] 搜索不存在的关键词显示空结果提示，不崩溃。
- [ ] 直接搜索结果只展示资料与“待社团确认”，不得伪装成由偏好问卷产生的匹配理由。

### 候选边界、状态与顺序

- [ ] 只选 1 个时不能开始比较，并给出还需选择 1 个的明确反馈。
- [ ] 选择 2 个后可以比较；选择 3 个后仍可比较。
- [ ] 已有 3 个候选时尝试添加第 4 个，选择保持为原 3 个并显示“最多选择 3 个”。
- [ ] 在 Top 6 与搜索模式之间切换时，已选候选和底部选择托盘保持一致。
- [ ] 从比较页返回选择页时，问卷答案、Top 6、搜索条件所在页面状态和已选候选在当前会话内合理保留。
- [ ] 比较页列顺序与用户选择 2–3 个社团的先后顺序一致。

### 比较与结果

- [ ] 比较页展示每周投入、活动频率、普通成员职责、互动氛围四维。
- [ ] 候选值不同的行有清晰高亮，关键差异摘要只列真实不同项，并能对应到具体社团。
- [ ] 有问卷答案时可以看到候选的匹配理由；直接搜索进入比较时不出现伪造的匹配理由。
- [ ] 结果页提供每个已选社团的“更倾向”选项，以及“先都不加”。
- [ ] 未选择倾向直接确认时出现明确提示。
- [ ] 选择某社团后生成中文预期卡；卡片不暴露 `project`、`regular` 等内部英文枚举。
- [ ] 选择“先都不加”后生成对应中文行动卡，且不暗示用户必须加入。
- [ ] 确认后恰好展示 3 条可执行的现场追问，问题与候选差异、当前倾向/偏好或待确认状态有关。
- [ ] 所有候选资料、预期卡或相关区域明确提示“待社团确认”，线下加入说明可见。

### 返回、重开与系统边界

- [ ] 结果页“返回调整”能回到候选选择，当前会话中的答题与选择仍可继续修改。
- [ ] “重新开始”回到首页并清空答题、Top 6、候选和倾向。
- [ ] 彻底关闭并重开小程序后不承诺恢复上轮状态；界面不得宣称已持久保存。
- [ ] 断网后四页主链路、搜索、匹配、比较和结果卡仍可使用；调试器 Network 无业务网络请求。
- [ ] 未出现登录、云同步、远程数据库、AI/LLM、站内扫码、加群或报名能力。
