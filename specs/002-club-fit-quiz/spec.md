# 002 — 社团倾向小测试规格

> 本规格描述面向演示原型的固定规则偏好测试，不是专业心理测评，也不使用 AI。

## 用户目标

作为暂时没有明确候选社团的新生，我希望通过简短问题缩小范围，再把推荐结果放进现有对照表继续比较。

## 流程

`pages/index` → `pages/quiz` → `pages/quiz-result` → `pages/compare` → `pages/result`

首页同时保留原有直接选择路径：

`pages/index` → `pages/list` → `pages/compare` → `pages/result`

## 验收条件

1. 首页提供“开始倾向测试”和“直接选择社团比较”两个入口。
2. 测试共 8 道单选题，逐题展示当前进度。
3. 未选择答案时不能进入下一题，并显示明确提示。
4. 用户可以返回上一题修改答案。
5. 相同答案始终产生相同顺序的推荐结果。
6. 全部 98 个社团都参与匹配，并且每个社团都有明确的兴趣领域映射。
7. 结果页展示 3 个不同的候选社团及可解释的匹配理由。
8. 结果只引用 `data/clubs.js` 中存在的社团，不生成不存在的社团或信息。
9. 用户可以把 3 个推荐结果带入现有比较页。
10. 用户可以重新测试。
11. 首页提示“演示原型，社团实际信息以现场介绍为准。”
12. 结果页提示“测试结果仅供参考。”
13. 原有直接选择、2–3 个约束、四维比较和最终倾向流程保持可用。

## 固定评分原则

- 第 1 题匹配兴趣领域，权重最高。
- 其余 7 题分别匹配时间投入、活动强度、协作方式、公开表达、实践程度、竞技程度和安排规律性。
- 社团兴趣领域由仓库内显式映射给出；其他特征根据当前结构化字段通过固定关键词和时间规则生成。
- 分数相同时按 `data/clubs.js` 中的稳定顺序排列。
- 结果不宣称心理类型、科学准确率或加入成功率。

## 非目标

- 不创建 MBTI 类型或人格标签。
- 不使用登录、后端、远程请求、AI、随机推荐或新 npm 依赖。
- 不持久化答案，不收集个人信息。
- 不在小程序内完成扫码、加群或报名。
