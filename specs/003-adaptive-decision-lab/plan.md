## 2. 宪法式门槛

1. **本地优先**：完整主路径断网可运行。
2. **证据先于结论**：字段值没有合格 evidence 时按未知处理。
3. **硬约束与偏好分离**：三值门禁不进入 0–100 软偏好公式。
4. **可解释且确定**：相同输入必须得到相同题序、分数、排序和理由。
5. **用户保有决定权**：系统缩小范围、暴露权衡，不替用户报名或下最终结论。
6. **规格先行**：行为、字段或路由变化先更新 003 与数据契约；破坏上述门槛须新增 ADR。

设计前后检查：当前方案满足六项门槛。若实施中引入持久化、远程数据、统计模型或新页面，本计划自动失效，必须重新评审。

## 3. 目标架构

### 3.1 分层

1. **资料层**：`data/clubs.js` 保存社团基本资料、决策字段和字段级 evidence；题库使用独立本地模块。
2. **规则层**：
   - 题目规划器：8 题核心 + 最多 6 题自适应；
   - 硬约束引擎：逐项输出 `PASS/FAIL/UNKNOWN`；
   - 偏好引擎：六维 0–100 匹配与独立证据置信度；
   - 压力引擎：逐情境输出 `RESILIENT/AT_RISK/UNKNOWN`；
   - 报告生成器：聚合快照和最多 5 条核实问题。
3. **会话层**：集中维护版本化 `decisionSession`，提供重置、失效传播和前置状态守卫。
4. **页面层**：八页只编排展示和用户事件，不复制评分、证据或压力规则。

### 3.2 数据流

`assessment answers`
→ 编译 `decisionProfile`
→ 硬约束三值判定
→ 六维偏好匹配 + 证据置信度
→ `recommend / library / club`
→ `selection` 形成 2–3 候选快照
→ `compare` 差异 + 压力测试
→ `result` 非绑定性报告。

任何评估答案变化都会递增 `calculationVersion`。下游对象携带生成版本；版本不一致时页面只能提示重算，不能展示旧结论。

## 4. 页面职责与导航

- `index`：创建或重置会话；说明 8–14 题、证据等级和线下边界；以三个入口区分专业测评、条件筛选和直接浏览，其中后两者复用 `library`。
- `assessment`：呈现一题、写答案、调用题目规划器；不自行决定下一题。
- `recommend`：分开展示“可继续了解”和“被排除”；提供解释展开、详情和资料库入口。
- `library`：本地搜索与多维条件交集筛选；条件入口默认展开筛选器，直接浏览入口保持默认浏览态；不制造匹配理由。
- `club`：字段值与 evidence 同屏；加入/移除候选。
- `selection`：统一候选篮，显示 2–3 边界、来源、过期状态和证据缺口。
- `compare`：同维表格、差异和标准/相关压力情境；要求有效候选与当前版本。
- `result`：记录三种非绑定性结论，生成报告和核实问题；提供返回调整与重开。

目标 `app.json` 仅注册上述八页。迁移期 `scripts/validate-project.js` 可接受旧四页、目标八页或仅由批准路由组成的有序中间态；迁移完成任务必须收紧为目标八页精确断言。

## 5. 核心算法设计

### 5.1 自适应选题

- 题库记录 `id`、覆盖维度、是否核心、稳定顺序、可消除的未知、候选区分函数和答案契约。
- 先按稳定顺序完成 8 道核心题。
- 追加题优先级为：消除硬约束未知 > 区分前 3 候选 > 补最低置信软维度。
- 前 3 相邻偏好分差均至少 8、硬约束用户输入完整且六维置信达阈值时提前停止；否则最多 14 题。
- 修改答案后，根据依赖图删除受影响的后续题和答案，再确定性重放。

### 5.2 硬约束

每项判定保存 `constraintId`、用户要求、候选字段、结果、reason、evidenceRefs。判定顺序：

1. 值或 evidence 缺失 → `UNKNOWN`；
2. evidence 来源是 `prototype-inferred` → `UNKNOWN`；
3. evidence 不具备硬通过资格 → 相符时 `UNKNOWN`，明确冲突且来源可用于否定时按契约判定；
4. 合格事实明确冲突 → `FAIL`；
5. 合格事实明确满足 → `PASS`。

候选总体状态：任一 `FAIL` 即 `FAIL`；无失败但任一未知即 `UNKNOWN`；全部通过才是 `PASS`。

### 5.3 多准则偏好匹配

六维权重归一化为总和 100。每维兼容度由显式映射表得到 0、25、50、75 或 100；未知字段固定使用中性计算值 50，并记录 `isUnknown: true`。总分为加权平均的四舍五入整数。

证据置信度独立计算：A=100、B=80、C=60、D=30、U=0；按本次实际使用的六维权重加权。它只表达“这个分数依赖的资料有多可靠”，不改变用户偏好分。

### 5.4 压力测试

压力情境是显式规则，不改变原资料。每条规则声明需要字段、阈值、结果和解释模板。任一必需字段未知或只有原型推断时输出 `UNKNOWN`；明确越界输出 `AT_RISK`；拥有合格证据且仍在用户容忍范围内输出 `RESILIENT`。

## 8. 风险与控制

- **原型资料误导**：字段级 evidence、逐项标签、硬通过资格限制。
- **分数伪精确**：整数分与逐维贡献并列显示，置信度分离，禁止概率措辞。
- **自适应逻辑不可复现**：稳定题序、无随机数、快照测试。
- **状态串页污染**：版本号、前置守卫、统一失效传播。
- **八页造成范围膨胀**：每页单一职责；不新增账户、云、报名或第九页。
- **旧流程回归**：001/002 保留为 checkpoint 测试基线，迁移完成前持续运行。
