# 项目主计划 — club-compare-miniprogram

> 来源说明：本文档于 2026-07-21 根据仓库现有代码、`README.md`、`AGENTS.md`、
> `validation-checklist.md` 和 `fix-log.md` 整理，用于建立最小协作基线；它不是对缺失课程文档的原文恢复。

## 产品问题

大学新生在加入社团前通常只能看到宣传口号，难以快速了解真实活动、时间投入、普通成员职责和社团氛围。

## 产品目标

提供一个本地优先的微信小程序原型，让用户选择 2–3 个候选社团，并排比较四个固定维度，最后记录：

- 更倾向某个已选社团；或
- 先都不加。

真实扫码、加群、联系负责人和报名均在线下渠道完成。

## 当前基线

当前 walking skeleton 已包含四页流程：

`pages/index` → `pages/list` → `pages/compare` → `pages/result`

数据来自 `data/clubs.js`，选择和倾向仅保存在当前会话的 `globalData` 中。正式验收标准见
`specs/001-club-compare-flow/spec.md`，数据字段见 `docs/data-contract.md`。

## 非目标

未经单独批准，不增加：

- 登录、云同步、微信云开发或远程数据库；
- 后端 API、实时 AI/LLM 推荐；
- 支付、数据分析 SDK；
- 小程序内扫码、加群或报名；
- 新 npm 依赖和无关模板页面。

## 阶段

1. **可运行原型**：现有四页主流程和本地样例数据。
2. **协作基线**：仓库内规格、数据契约、Issue/PR 模板和可复现验证步骤。
3. **本地数据扩展**：将本地样例扩展为 98 个社团；分类字段仍需单独批准。
4. **社团倾向测试**：通过 8 道题和固定规则推荐 3 个候选，再进入现有比较流程。
5. **候选分类筛选**：依赖已经冻结的分类字段，不改变 2–3 个选择约束。
6. **候选比较后支持**：提供返回修改选择和缺失信息提醒，不使用 AI。

阶段 3–4 已由 Issue #2–#3 推进；阶段 5–6 仍是候选 backlog，不代表已经批准开发。详见
`specs/001-club-compare-flow/remaining-backlog.md`。

## 协作门槛

- 不直接在 `main` 开发；一个 Issue 对应一个负责人、一个分支和一个 PR。
- 共享数据结构、`app.json`、`app.js` 和 `data/clubs.js` 由单一负责人顺序修改。
- 行为变化先核对规格；数据结构变化先更新数据契约。
- PR 前运行静态检查，并由人工在微信开发者工具中走查相关流程。
- 人工查看 `git diff` 后才允许 commit；PR 由另一名成员 Review 后合并。
